Адресация позволяет узнать, кому адресован тот или иной конкретный пакет.

Протокол использует однобайтовую адресацию в виде типа `ruavp_address_t`:
```c
typedef uint8_t ruavp_address_t;
```

Первые 4 бита типа - номер устройства, которому адресован пакет.
Последние 4 бита - класс устройства.

##### Классы
Всего в протоколе возможно 14 классов. Диапазоны `0x00`-`0x0F` и `0xF0`-`0xFF` зарезервированны для внутреннего использования.

В каждом классе может быть до 15 устройств.

Текущая версия протокола поддерживает следующие классы:
```c
enum ruavp_class_t
{
  CLASS_LAND = 1 << 4,
  CLASS_HELI = 2 << 4,
  CLASS_HELIHW = 3 << 4,
  CLASS_EXT = 4 << 4,
};

enum ruavp_class_max_t
{
  CLASS_LAND_MAX = 7,
  CLASS_HELI_MAX = 7,
  CLASS_HELIHW_MAX = 7,
  CLASS_EXT_MAX = 7,
};
```

Перечисление `ruavp_class_t` определяет идентификаторы для классов. Перечисление `ruavp_class_max_t` определяет максимальное количество устройств в классе.

Например, чтобы адресовать устройство под номером 2 в классе `HELIHW`:
```c
ruavp_address_t addr = CLASS_HELIHW | 2;
```

Номер 0 используется для адресации всех устройств в классе. Сообщение вида:
```c
ruavp_address_t addr = CLASS_LAND;
```
будет отправлено на все устройства класса `LAND`.

Номера устройств присваиваются на этапе компиляции и не могут быть изменены во времени выполнения.

> [!note]
> Если использовать 0 в качестве собственного адреса, система будет получать *все* входящие пакеты. Это можно использовать для реализации приложения, которое будет записывать все данные в системе.
